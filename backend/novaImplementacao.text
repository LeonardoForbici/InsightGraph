## 7.1 — Persistent Embedding Index (SQLite + FAISS-like)
No backend, substituir rag_index.json por:

Novo arquivo: rag_store.py
- SQLite table: node_embeddings(key, blob, embedding BLOB, updated_at)
- Usar numpy arrays serialized como bytes (sem FAISS)
- Índice em memória: carregar ao startup, atualizar incremental
- Similaridade coseno: implementação pura numpy (sem deps extras)
- Threshold de relevância: só retornar se similarity > 0.65
- Cache LRU para as 1000 consultas mais recentes
- Background task: re-embedar nós alterados após cada scan

## 7.2 — Context-Aware RAG
Melhorar /api/ask para:
- Detectar automaticamente se pergunta é sobre:
  * Nó específico → buscar vizinhos + fluxo de dados
  * Padrão arquitetural → buscar todos os exemplos do padrão
  * Impacto de mudança → pré-carregar blast radius
  * Análise de segurança → incluir vulnerabilidades CodeQL

- Multi-hop context: se pergunta menciona "transação de pagamento",
  buscar todos os nós no path da transação via transaction view

- Citar fontes no response: "Esta resposta usa contexto de: 
  [PaymentService.java, OrderRepository.java, ...]"
